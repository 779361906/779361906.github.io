Unity支持使用js，c#，boo三种语言作为脚本，推荐使用js。
使用js脚本时需要注意以下几点：

  1. Unity支持的js语法和普通的js是一致的。
  2. 使用js脚本，脚本必须与对象绑定才能生效。
  3. 应注意脚本变量的生命周期，和通用的js有所不同，下面会具体介绍。

具体使用方法：

  1. 创建js脚本,鼠标在工程目录下右击，选择“Create” —> “Javascript”.生成js脚本，需要命名：
  2. 编辑js脚本，选择目录下的js脚本文件，按回车。进入Unity提供的js编辑器，支持语法高亮。如果不喜欢自带的编辑器，可使用系统自带的文件管理器进入文件目录，使用你喜欢的编辑器对js脚本进行编辑。
  3. 绑定对象。工程目录下选中js脚本文件，将它拖到你要绑定的对象上即可。此时点击对象查看对象的“Inspector”面板可以看到有一个js脚本名的标签栏，可在这里选择js脚本的使用，默认是使用的。
  4. js生命周期的注意事项。请参考unity 脚本 function 生命周期 MonoBehaviour Mono行为 和
    MonoBehaviour的前世今生

主要可以理解为：

OnAwake 方法: 当脚本实例被载入时Awake被调用

OnStart 方法: Start仅在Update函数第一次被调用前调用。

OnUupdate 方法 ：渲染一帧之前被调用。这里是大部分游戏行为代码被执行的地方，除了物理代码

OnGUI 方法: 渲染和处理GUI事件时调用 这里不是 没帧都调用哦

FixedUpdate 方法: 这个函数在每个物理时间步被调用一次。这是处理基于物理游戏行为的地方。

LateUpdat 方法 :是在所有Update函数调用后被调用。这可用于调整脚本执行顺序。例如:当物体在Update里移动时，跟随物体的相机可以在LateUpdate里实现

Reset 方法 ： 重置为默认值

OnDisable 方法: 当对象变为不可用或非激活状态时此函数被调用。

OnDestroy 方法:当MonoBehaviour将被销毁时，这个函数被调用
